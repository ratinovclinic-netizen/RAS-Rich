# Безопасная установка webhook Bitrix24

1. Перегенерировать webhook, если его адрес хотя бы один раз был отправлен в чат, письмо или показан на незакрытом скриншоте.
2. Не отправлять новый адрес разработчику в переписке и не вставлять его в исходный код.
3. Открыть файл `.env.local` в корне проекта.
4. Вставить новый адрес после `BITRIX24_WEBHOOK_URL=` без кавычек.
5. Сохранить файл. Он исключён из Git и не попадёт в репозиторий.
6. Для опубликованного сайта добавить значение `BITRIX24_WEBHOOK_URL` в защищённые серверные переменные проекта Sites.
7. После настройки открыть защищённый маршрут `/api/integrations/bitrix/inventory` под учётной записью руководителя.

Маршрут выполняет только разрешённые методы чтения, не возвращает секрет webhook и не кэширует ответ.

## Требования к пользователю Bitrix24

- отдельная техническая учётная запись;
- просмотр необходимых сделок, воронок, полей и активностей;
- запрет изменения и удаления CRM-объектов;
- отсутствие административных прав;
- webhook с минимально необходимыми разделами доступа.
